下一段：普通 AI 冲榜为什么容易失控

Why It Matters

普通 AI 冲榜容易失控，因为它没有决策系统。

评价不可信

本地验证、public leaderboard、hidden official、smoke test 混在一起，弱证据容易被当成强结论。

路线不可比

看到一个新模型就适配，缺少任务范式分类，也缺少方法角色管理。

提交被污染

不值得交的结果也被打包，浪费提交机会，还会污染后续路线判断。

Workflow Map

先锚定评价，再扫方法池，再做结构创新，最后才冲提交。

这不是一个 prompt，而是一组 Agent Harness 风格的阶段约束、spec、scoreboard、decision record 和 submission record。

L0
规则与验证锚定
规则、数据、指标、本地验证、baseline、提交格式。
L1
强范式扫榜
任务范式分类，每个范式挑 1-2 个强方法做本地适配。
L2
问题驱动结构创新
现象 -> 问题 -> 瓶颈 -> 机制 -> 结构改写。
L3
竞赛工程优化
fold、pseudo-label、TTA、postprocess、ensemble。
Gate
提交意图门控
只有 package_for_submission 才允许生成提交包。
Loop
榜单反馈闭环
记录 rank、score、delta、风险和下一步动作。

L1 Is The Core

先做范式分类，再让强方法一圈跑通。

L1 宏观方法池 spec

先判断任务是新任务还是经典任务，再调研近年 SOTA 范式。排序按综合打分：上限、适配成本、代码成熟度、算力、规则风险。

每个方法一张适配 spec

本地适配不是随手改代码。每条路线必须明确输入输出、权重来源、验证协议、provenance、失败条件和晋级门槛。

Problem-Driven Innovation

L2 不是加 trick，而是把现象抽象成问题。

Phenomenon强 baseline 到瓶颈，某类误差稳定存在。
Problem定义当前任务真正没被解决的问题。
Bottleneck定位结构、数据、指标或几何机制上的限制。
Mechanism解释为什么某个结构改写可能有效。
Spec写结构 spec、可证伪实验和晋级门槛。

强制门槛

Problem Definition Card 必须先于 Innovation Card。

Innovation Card 必须先于 Structural Spec。

L2 只处理结构创新。TTA、阈值、后处理、ensemble 属于 L3。

Tabbit As Context Surface

Tabbit 可以作为现场上下文入口，但 workflow 不依赖某个工具。

现场可以把截图和 markdown 作为上下文交给 Tabbit，让它按 Competition Workflow Skill 输出路线决策和提交门控。

Closing

AI 的价值不是替我冲动提交，而是帮我维护可验证、可追溯、可停止的冲榜决策系统。

结果可验证真实榜单截图和分数证据。
流程可复用L0-L3、gate、feedback 可迁移。
决策可追溯scoreboard 和 provenance 记录每次判断。
专业性强知道什么时候推进，也知道什么时候停止。
适合高手服务快速路线决策，不是泛泛聊天。
